# Release v0.1.0 — 2026-04-28

Summary

pic16cc v0.1.0 marks the Phase 8 milestone: source-level typedef/enum/struct support, positional aggregate initializers, and improved cast/member semantics on top of the Phase 7 optimization layer and Phase 6 ISR model.

Highlights

- Frontend: `typedef`, `enum`, and named `struct` parsing and AST support.
- Frontend: positional brace initializers for arrays and structs; globals pre-materialized to bytes.
- Frontend: explicit cast syntax with Phase 8 restrictions and diagnostics.
- Frontend: member access operators `.` and `->` lowered to pointer+offset+deref.
- Semantic: enum constants as compile-time symbols; initializer flattening for locals and globals.
- IR/Backend: aggregate global bytes emitted at startup; member access lowering preserved across lowering passes.
- Tests: integration tests added covering Phase 8 features; CI test suite green locally.
- Docs: design notes and example code added under `docs/` and `examples/`.

Important Notes and Limitations

- Structs use a packed layout (no padding) in this release.
- Nested aggregates and designated initializers are intentionally rejected in Phase 8.
- Enum representation is fixed to 16-bit for this release.
- Pointer-to-pointer types, multidimensional arrays, and source-level function pointers remain unsupported.
- Phase 4 ABI and Phase 6 ISR constraints are preserved.

Validation

- Local verification: `cargo test --all` (all tests passed) and `cargo clippy --all-targets -- -D warnings` (no warnings).
- Examples in `examples/` compile successfully for the checked targets.

Upgrade / Migration Guidance

- Codebases depending on implicit struct padding or nested aggregate initializers will need small refactors.
- Replace nested initializers with manual field-by-field initialization or flatten structs where feasible.

Files Changed (high level)

- `src/frontend/*` — parser, AST, semantic analysis updates for Phase 8 types
- `src/ir/*` — minor lowering adjustments for member access and initializers
- `src/backend/*` — startup emission of global aggregate bytes, codegen adjustments
- `tests/compiler_pipeline.rs` — new integration tests for Phase 8
- `docs/*` — new Phase 8 docs and examples
